# AI 工作流与 Prompt 流程设计

## 总体架构

产品回答三个问题，对应三层 AI Pipeline：

```
Q1: 做什么内容？  →  Layer 1: 内容机会发现
Q2: 怎么做？      →  Layer 2: AI 辅助创作
Q3: 效果如何？    →  Layer 3: 归因反馈
```

每层内部是 **检索 → 推理 → 结构化输出** 的标准 AI 工作流节点，层与层之间通过数据管道串联成闭环。

---

## 一、Layer 1 内容机会发现 —— AI 工作流

### 1.1 整体流程

```
爬虫增量数据 → 清洗去噪 → Embedding → 向量库入库
                                    ↓
用户 Query → 混合检索(Dense+Sparse+RRF) → Top-K候选(50-80条)
                                    ↓
                              LLM-Rec 重排序 → Top-5推荐
```

### 1.2 Prompt 1：内容质量评分（训练数据筛选用）

**调用时机**：离线批处理，对爬取内容进行质量过滤，筛选 ≥3.5分 的样本进入 LoRA 训练集。

**Prompt：**

```
你是一个内容质量评审专家。请对以下种草/测评内容从五个维度打分（1-5分），并给出总分。

评分维度：
1. 信息密度：内容是否包含具体数据、细节、可操作信息？（空话套话=1分，干货密集=5分）
2. 情感感染力：内容是否能引起共鸣、建立信任感？（机械罗列=1分，读后有冲动=5分）
3. 结构清晰度：逻辑是否通顺、层次是否分明？（混乱跳跃=1分，一目了然=5分）
4. 原创性：观点或表达是否有独到之处？（洗稿拼接=1分，耳目一新=5分）
5. 行动引导力：看完后是否知道下一步该做什么？（无感=1分，立刻想试=5分）

[待评分内容]
标题：{title}
正文：{content}

输出要求：
- 严格按 JSON 格式输出
- 如果总分 < 3.5，在备注中简要说明"主要扣分维度"
- 不要输出任何 JSON 之外的内容

{
  "scores": {
    "info_density": <1-5>,
    "emotional_appeal": <1-5>,
    "structure": <1-5>,
    "originality": <1-5>,
    "call_to_action": <1-5>
  },
  "total": <平均分,保留1位小数>,
  "verdict": "pass" | "fail",
  "note": "<扣分原因，pass时为空字符串>"
}
```

### 1.3 Prompt 2：LLM-Rec 选题重排序与差异化推荐

**调用时机**：每日凌晨异步预计算，用户打开时即时展示。

**Prompt：**

```
你是一个内容策略分析师，专门为{domain}赛道的创作者提供选题建议。
你的信息来源仅限于下方提供的候选内容列表。不要编造列表中不存在的信息。

## 创作者画像
- 领域：{domain}
- 粉丝特征：{fan_profile_summary}
- 历史高互动内容特征：{historical_pattern}
- 近期已覆盖的话题（避免重复推荐）：{recent_topics}

## 候选内容池（共{count}条）
{candidate_list}
每条包含：标题、标签、互动量(点赞/收藏/评论)、发布时间、内容类型

## 任务
从候选池中选出 Top-5 最适合该创作者的选题方向，并对每个选题给出差异化切入角度。

## 排序原则
1. 与创作者粉丝画像的匹配度（权重最高）
2. 话题的新鲜度（优先近7天、竞品尚未大量覆盖的方向）
3. 互动信号质量（收藏率 > 点赞率 的"高价值内容"加分）
4. 差异化空间（能用创作者独特视角切入的 > 已经被写烂的通用选题）
5. 与创作者历史内容的互补性（填补类型空白 > 重复已有类型）

## 重要约束
- 如果候选池整体质量不高（多数内容同质化严重或时效性差），你必须诚实告知，不要强行推荐
- 每个推荐必须给出一个具体的、可执行的差异化切入角度，避免"从XX角度分析"这种空话

## 输出格式（严格 JSON，不要输出其他内容）
{
  "overall_assessment": "<一句话总结当前赛道的内容机会状况>",
  "recommendations": [
    {
      "rank": 1,
      "topic": "<选题方向>",
      "reason": "<一句话推荐理由，含数据引用>",
      "differentiation_angle": "<具体的差异化切入角度，必须是可执行的创作建议>",
      "target_fan_segment": "<最适合哪类粉丝群体>",
      "estimated_effort": "low" | "medium" | "high",
      "reference_content_id": "<候选池中作为参考的内容ID>"
    }
  ],
  "low_quality_warning": "<候选池质量不佳时填写，否则为null>"
}
```

### 1.4 Prompt 3：粉丝画像生成

**调用时机**：用户首次接入时，基于平台后台数据生成。

**Prompt：**

```
你是一个用户画像分析师。请根据以下粉丝群体的统计数据，生成一段简洁的粉丝画像描述。

## 输入数据
- 年龄分布：{age_distribution}
- 性别比例：{gender_ratio}
- 地域分布(TOP5)：{top_regions}
- 活跃时段：{active_hours}
- 近期互动高频词（来自评论/收藏内容）：{top_keywords}
- 历史互动内容的共同特征：{content_features}

## 任务
将粉丝划分为 2-3 个主要群体，每个群体用一句"人话"描述其特征和内容偏好。

## 输出格式（严格 JSON）
{
  "segments": [
    {
      "name": "<群体昵称，如'成分党''场景派'等，简洁有记忆点>",
      "proportion": "<占比，如'约40%'>",
      "description": "<一句话描述：这群人关心什么、为什么消费、喜欢看什么类型的内容>",
      "content_preference": "<内容类型偏好>",
      "decision_factor": "<影响其消费决策的最关键因素：价格/品牌/口碑/颜值/成分>"
    }
  ],
  "overall_summary": "<一段话总结该账号的整体粉丝画像,≤100字>"
}
```

## 二、Layer 2 AI 辅助创作 —— Prompt Pipeline

### 2.1 五步生成 Pipeline 总览

```
输入：Layer1 的选题+差异化角度+目标粉丝画像
  ↓
Step 1: 大纲生成 → 3-5节点大纲 + 每节点关键信息
  ↓  [用户可编辑调整]
Step 2: 分段扩写 → 每段独立生成(≤500 tokens)，完整草稿
  ↓
Step 3: 风格润色 → 注入风格变量 + 品牌语音 → 风格优化版
  ↓
Step 4: 质量自检 → LLM自评 + Embedding相似度检测 → 通过/重写
  ↓  [不通过则回Step2]
Step 5: 多平台适配 → 小红书版/抖音版/B站版
  ↓
输出：三平台版本内容
```

### 2.2 Prompt 4：大纲生成（Step 1）

**调用时机**：用户确认选题后，第一步生成。

**Prompt：**

```
你是一个专业的{domain}内容创作者。请为以下选题生成一个内容大纲。

## 任务信息
- 选题：{topic}
- 差异化切入角度：{differentiation_angle}
- 目标读者：{target_fan_segment}
- 预期内容类型：{content_type}（教程/测评/场景故事/清单/对比横评）
- 品牌调性关键词（如有）：{brand_tone_keywords}

## 要求
1. 生成 3-5 个大纲节点，逻辑递进
2. 每个节点标注预估字数（总计控制在 {target_word_count} 字以内）
3. 每个节点列出 2-3 个"必须覆盖的关键信息点"
4. 开头节点必须包含一个"钩子"设计（3秒内抓住读者注意力的方案）
5. 结尾节点必须包含行动引导（让读者知道看完后该做什么）

## 风格约束
- {style_instruction}（从风格池采样注入，如"采用专业测评风，数据驱动，成分分析，理性克制"）
- 避免使用以下词汇：{banned_words}（如有品牌禁用词列表）
- 禁止使用"众所周知""毋庸置疑""在当今社会"等空话开头

## 输出格式（严格 JSON）
{
  "hook_strategy": "<开头钩子方案，一句话>",
  "nodes": [
    {
      "order": 1,
      "title": "<节点标题>",
      "estimated_words": <字数>,
      "key_points": ["<要点1>", "<要点2>", "<要点3>"],
      "purpose": "<这个节点在整篇内容中的作用>"
    }
  ],
  "call_to_action": "<结尾行动引导>",
  "total_estimated_words": <总预估字数>
}
```

### 2.3 Prompt 5：分段扩写（Step 2）

**调用时机**：用户确认大纲后，每段独立调用一次。

**Prompt：**

```
你是一个{domain}领域的专业内容写手。请根据以下信息扩写内容的一个段落。

## 上下文
- 全文主题：{topic}
- 差异化角度：{differentiation_angle}
- 目标读者：{target_fan_segment}
- 上一段末句：{previous_paragraph_last_sentence}

## 本段要求
- 节点标题：{node_title}
- 本段在全文中的作用：{node_purpose}
- 必须覆盖的关键信息点：{key_points}
- 目标字数：{estimated_words}字（±20%可接受）

## 写作风格
- {style_instruction}
- 使用目标读者熟悉的语言和场景
- 每个抽象观点后面跟一个具体例子或数据
- 段与段之间用过渡句衔接（首句承接上段末句）

## 禁止事项
- 禁止使用品牌禁用词：{banned_words}
- 禁止编造数据（如果某个信息点没有可靠来源，标注"建议补充XX数据"）
- 禁止输出超过 800 字（长段影响阅读体验，超长内容应拆分到下一个节点）

## 输出格式（严格 JSON）
{
  "content": "<扩写后的段落文本>",
  "last_sentence": "<本段末句，用于下一段承接>",
  "data_placeholders": ["<需要用户手动补充的数据点位，如无则为空数组>"],
  "word_count": <实际字数>
}
```

### 2.4 Prompt 6：风格润色（Step 3）

**调用时机**：扩写完成后，注入风格变量+粉丝画像适配。

**Prompt：**

```
你是一个内容润色编辑。请对以下草稿进行风格化润色，使其更贴合目标读者和预设风格。

## 输入
- 原始草稿：{draft_content}
- 预设风格：{style_instruction}
- 目标读者画像：{target_fan_segment_description}
- 品牌语音指南（如有）：{brand_voice_guide}

## 润色方向
1. 调整语气和用词，使其符合目标读者的阅读习惯：
   - 成分党 → 强化数据/成分/功效描述，使用准专业术语
   - 场景派 → 强化场景/体验/感受描述，使用生活化语言
   - 价格敏感型 → 强化性价比/对比/省钱信息
   - 品牌敏感型 → 强化品牌故事/调性/背书
2. 优化节奏：长句拆短、短句适当合并，提升阅读流畅度
3. 检查并修正"AI味"：
   - "在当今社会""随着XX的发展""众所周知" → 全部删除
   - "不仅...而且...""一方面...另一方面..." → 简化
   - 过度使用"非常""十分""极其"等程度副词 → 删减50%
4. 保留原文的核心信息和逻辑结构，不改变事实表述

## 输出格式（严格 JSON）
{
  "polished_content": "<润色后的完整文本>",
  "changes_summary": ["<列出3-5条主要修改，每条一句话>"],
  "ai_tone_score": <1-5, 1=完全不像AI写的, 5=AI味浓重. 自评>
}
```

### 2.5 Prompt 7：质量自检（Step 4）

**调用时机**：润色完成后，作为发布前的最后一道关卡。

**Prompt：**

```
你是一个严格的内容质量审核员。请对以下内容进行审核打分。你唯一的职责是找出问题，不要为了"通过"而放水。

## 待审核内容
{content_to_review}

## 审核维度（每个维度1-5分，3分以下为不合格）

### 事实准确性
- 是否有明显的事实错误？
- 是否有编造的数据或引用？
- 是否有夸大或误导性表述？

### 逻辑连贯性
- 论点是否清晰？论证是否成立？
- 段落之间的过渡是否自然？
- 是否存在逻辑跳跃或自相矛盾？

### 风格一致性
- 整篇的语气和风格是否统一？
- 是否符合预设风格：{expected_style}？
- 是否符合品牌禁用词要求：{banned_words}？

### 原创性
- 是否有洗稿或过度借鉴的痕迹？（与训练数据中的常见表述高度雷同）
- 是否有独特的观点或表达？

### 商业合规
- 是否含有广告法禁止的绝对化用语（"最好""第一""唯一""100%"等）？
- 是否含有可能引发争议的比较或评价？
- 是否保留了必要的免责或说明？

## 输出格式（严格 JSON）
{
  "scores": {
    "factual_accuracy": <1-5>,
    "logic_coherence": <1-5>,
    "style_consistency": <1-5>,
    "originality": <1-5>,
    "compliance": <1-5>
  },
  "total": <平均分>,
  "verdict": "pass" | "rewrite" | "block",
  "issues": [
    {
      "severity": "critical" | "major" | "minor",
      "dimension": "<所属审核维度>",
      "location": "<问题在文中的位置（引用原句）>",
      "description": "<问题描述>",
      "suggestion": "<修改建议>"
    }
  ],
  "rewrite_focus": "<如需重写，给出重写的重点方向。pass或block时为空>"
}
```

**决策逻辑**：
- `total ≥ 3.5` 且无 critical issue → `pass`
- `2.5 ≤ total < 3.5` → `rewrite`，携带具体问题回 Step 2
- `total < 2.5` 或存在合规 critical issue → `block`，强制人工介入

### 2.6 Prompt 8：多平台适配（Step 5）

**调用时机**：质量自检通过后，同一内容生成三个平台版本。

**Prompt（三平台共用模板，通过 platform_config 区分）：**

```
你是一个多平台内容运营专家。请将以下原始内容适配为{platform}版本。

## 原始内容
{source_content}

## {platform} 平台规则
{platform_rules}

## 适配要求
1. 保留原始内容的核心信息和观点，不改变事实
2. 调整表达方式以符合{platform}用户的阅读习惯和平台调性
3. 如果原始内容长度不足，不要强行灌水；如果过长，精简约30%
4. 输出干净版本，不包含"以下是XX版本"等说明文字

## 输出格式（严格 JSON）
{
  "title": "<适配后的标题>",
  "content": "<适配后的正文>",
  "hashtags": ["<推荐3-5个标签>"],
  "cover_text": "<建议封面文字，≤15字>",
  "platform_tips": "<一条针对该平台的发布建议（如最佳发布时间、互动引导策略）>"
}
```

**platform_rules 配置表：**

| 平台 | platform_rules |
|------|---------------|
| 小红书 | 目标读者为18-35岁女性为主。内容要求：知识密度高、审美感强、适当使用emoji分段、首图即关键信息。正文风格：真实感 > 精致感，口语化但信息不水。标签 ≤10个。禁止过度营销话术。 |
| 抖音 | 目标读者为全年龄段短视频用户。内容要求：开头3秒必须有强钩子、句子短（≤20字/句）、节奏快、情绪强。正文风格：直接、有冲击力、口语化。善用热门BGM和挑战话题关联。 |
| B站 | 目标读者为18-30岁深度内容消费者。内容要求：允许深度展开、可以适度玩梗、预留弹幕互动空间。正文风格：专业但不端着，可以吐槽、有态度。善用"省流""总结"等B站特色结构。 |

## 三、Layer 3 归因反馈 —— AI 工作流

### 3.1 整体流程

```
全链路数据采集 → 数据清洗 → 贝叶斯归因模型(特征权重推断)
                                    ↓
                          归因结果结构化(JSON)
                                    ↓
                          LLM 周报生成 → 用户可读报告
                                    ↓
                          三级预警判定 → 实时推送/周报标注
```

### 3.2 Prompt 9：贝叶斯归因结果 → LLM 周报生成

**调用时机**：每周一 08:00 自动生成。

**Prompt：**

```
你是品牌的内容策略分析师。你的唯一信息来源是下方提供的数据。
如果数据不足以支撑某个结论，你必须诚实地说"数据不足，待观察"。
不要编造数据中没有的信息。

## 报告周期
{start_date} - {end_date}（第{week_number}周）

## 账号信息
- 账号名称：{account_name}
- 领域：{domain}
- 本周发布内容数：{total_posts}
- 累计可分析内容数：{cumulative_posts}

## 数据输入（JSON格式）
{attribution_data_json}
注意：每个数据点携带 {值, 样本量N, 95%置信区间, 来源}，置信区间宽的结论意味着不确定性高。

## 你的任务
基于以上数据生成一份周报。你需要：

1. 先判断每个数据的信度：
   - N > 100 且置信区间窄 → 高信度，可以下确定性结论
   - 30 < N ≤ 100 → 中信度，结论需附带"建议继续观察"
   - N ≤ 30 → 低信度，禁止下确定性结论

2. 输出以下结构的报告：

## 报告输出格式

### 1. 关键数字（3个核心指标）
- 📈 内容→转化率：{从数据中提取}，环比 △{变化百分比}
- 🏆 最佳内容："{标题}"，贡献了约 {百分比} 的转化
- ⚠️ 预警信号：{有则具体描述，无则写"本周无预警"}

### 2. 高转化内容模式（最多3个）
对每个模式：
- 模式特征描述（话题类型 + 内容形式 + 情感策略 + 发布时间）
- 数据支撑（转化贡献度 + 置信度标注）
- 行动建议："建议下周将此类内容比例提升至 X%"

### 3. 低效内容模式（最多2个）
- 模式特征 + 数据支撑
- 行动建议："建议暂停或调整为 XX 方向"
- 如果数据不够（N<30），明确说"数据不足以判断，暂时保持观察"

### 4. 下周行动建议（按预估效果排序，3-5条）
每条建议格式：
- 建议标题
- 理由（数据依据）
- 预估效果（保守估计）

### 5. 内容机会雷达（来自Layer1）
- 近7天你所在赛道新出现的内容趋势
- 值得关注的竞品策略变化
```

## 五、Prompt 工程策略总结

### 5.1 设计原则

| 原则 | 说明 | 应用 |
|------|------|------|
| **角色锚定** | 每个 Prompt 开头先设定明确角色和职责边界 | 所有 Prompt 首段 |
| **信源锁定** | 禁止 LLM 使用训练数据中的"常识"替代输入数据 | 归因报告、选题推荐 |
| **结构化输出** | 所有输出强制 JSON 格式，便于前端渲染和下游管道消费 | 所有 Prompt |
| **不确定性标注** | 数据不足时要求 LLM 显式标注"待验证"而非强行下结论 | Layer 3 归因报告 |
| **风格注入机制** | 不硬编码风格参数，而是通过"风格池采样+Prompt注入"实现多样性 | Layer 2 生成 |
| **多层防御** | 质量自检(LLM自评) + Embedding相似度检测(算法) + 人工最终审核 | Layer 2 Step 4 |
| **Token预算控制** | 每个 Prompt 右上方标注预估 Token 消耗，MVP阶段严格控制成本 | 全流程 |

### 5.2 风格池设计（防止内容同质化的关键）

每次生成从风格池随机采样 2 个风格，分别独立生成，用户选择或混合：

```
风格池（6个预设 + N个品牌自定义）：
1. 专业测评风 —— 数据驱动，成分分析，理性克制
2. 场景故事风 —— 日常场景切入，情感先行，沉浸感
3. 清单干货风 —— 结构化要点罗列，效率导向，一目了然
4. 对比横评风 —— 多产品并行比较，优缺分明，购买决策辅助
5. 深度科普风 —— 原理解析，知识密度高，建立专业信任
6. 真实日记风 —— 第一人称体验记录，真实感强，拉近距离
N. 品牌专属风格 —— 从历史高互动内容提取的个性化写作特征（Prompt 10）
```

### 5.4 降级策略

当 API 不可用或成本超预算时，自动触发降级：

| 场景 | 标准方案 | 降级方案 |
|------|---------|---------|

| LoRA推理不可用 | 本地Qwen2.5-7B+LoRA | 降级为 Qwen2.5-1.5B+LoRA 或调用 DeepSeek-V3 API |

## 七、前端交互的 Prompt 透明化设计

产品中每个 AI 操作都向用户展示：
1. **做了什么**（如"正在分析近7天美妆赛道的238条热门内容..."）
2. **为什么这么做**（如"为了找到与你的粉丝画像最匹配的选题"）
3. **置信度**（如"数据积累3周后推荐精准度将显著提升"）

这种设计不是技术细节的暴露，而是建立用户对 AI 输出的信任——尤其在 Layer 3 归因报告中，置信区间标签直接决定了用户是否会根据建议调整策略。
